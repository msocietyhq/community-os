# Product Requirements Documents (PRDs)

## Template

When creating PRDs, use this structure:

# PRD: Title

**Status**: draft | review | approved | in-progress | completed
**Author**: name
**Date**: YYYY-MM-DD
**Target Release**: description or date

## Problem Statement
What problem are we solving? Who has this problem?

## Goals
- Goal 1
- Goal 2

## Non-Goals
- What we're explicitly NOT doing

## User Stories
As a [role], I want to [action] so that [benefit].

## Requirements

### Must Have (P0)
- ...

### Should Have (P1)
- ...

### Nice to Have (P2)
- ...

## Technical Approach
High-level technical design.

## Success Metrics
How we'll measure if this was successful.

## Open Questions
Unresolved questions that need answers.
