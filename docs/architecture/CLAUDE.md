# Architecture Documents

## Template

When creating architecture documents, use this structure:

### Title

**Status**: draft | review | approved
**Last Updated**: YYYY-MM-DD

#### Overview
High-level description of the system or component.

#### Context
Why this architecture exists, what problem it solves.

#### Components
Description of each component and its responsibilities.

#### Data Flow
How data moves through the system.

#### Dependencies
External services, libraries, and infrastructure.

#### Trade-offs
What was considered and why this approach was chosen.

#### Open Questions
Unresolved architectural questions.
