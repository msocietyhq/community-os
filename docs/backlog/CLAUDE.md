# Backlog

## Priority Levels

| Level | Label | Description |
|-------|-------|-------------|
| P0 | Critical | Must be done for MVP launch |
| P1 | Important | Should be done soon after MVP |
| P2 | Nice to Have | Do when capacity allows |
| P3 | Future | Tracked for future consideration |

## Status Values

| Status | Description |
|--------|-------------|
| `todo` | Not started |
| `in-progress` | Currently being worked on |
| `blocked` | Waiting on something |
| `done` | Completed |

## Item Format

When adding backlog items, use this format:

### [P0] Item Title
**Status**: todo
**Assignee**: name or unassigned
**Notes**: any relevant context
